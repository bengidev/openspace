OpenSpace — Onboarding Feature Architecture

TCA-based layered flow: View → Reducer → Domain → Infrastructure

App Entry
View OnboardingContainerView
Root container. Creates StoreOf<OnboardingFlowFeature> with injected SwiftDataOnboardingRepository. Switches to HomeView when isFinished becomes true. Holds @AppStorage("appTheme") and resolves color palette.
Model OnboardingProgressEntity
SwiftData @Model. Fields: isCompleted, completedAt, lastPageIndex.
↓
TCA Business Logic
Parent Reducer OnboardingFlowFeature
State: currentPage, isFinished, pageDemo
Actions: onAppear, nextTapped, previousTapped, pageSelected, finishTapped, skipTapped
Effect: On appear → checks onboardingRepository.isOnboardingCompleted(). On finish → writes completion to repository.
↳ Child Scope — OnboardingPageDemoFeature
State: selectedPromptIndex, queuedPromptCount, reasoningLevel, pairingConfirmed
Actions: promptChipTapped, addQueuedPromptTapped, reasoningLevelChanged, pairingToggleTapped
Scoped via Scope(state: \.pageDemo, action: \.pageDemo)
Dependency onboardingRepository
Protocol OnboardingRepository with two methods. Injected into TCA dependency bag. Live = SwiftDataOnboardingRepository, Test/Preview = MockOnboardingRepository.
↓
Presentation Layer
View OnboardingView
Main layout. GeometryReader for responsive sizing. Background = palette.background + PixelGridBackground + DiagonalHatchPattern. Embeds top bar, feature page, and bottom navigation. Applies .sensoryFeedback(.selection, trigger: store.currentPage).
View OnboardingTopBar
Shows skip/close actions and theme toggle via appTheme binding.
View OnboardingFeaturePageView
Renders text content (eyebrow badge, headline, body) + terminal chrome card containing the interactive visual. Triggers entrance animation via runEntrance() per page change. Footer shows staggered OnboardingFeatureHighlight chips.
View OnboardingBottomNavigation
Pagination dots + Back / Continue or Enter OpenSpace button. Dots animate width on active page change.
↓
Visual Factory (PageVisualFactory)
Maps OnboardingPageModel → specific interactive visual View. Each visual receives demo state from OnboardingPageDemoFeature and sends actions back to the scoped child store.
SEC-01 encryptedPairing → EncryptedPairingVisual
AI-02 ideaStudio → IdeaStudioVisual
RUN-03 promptQueue → PromptQueueVisual
THK-04 reasoningControl → ReasoningControlVisual
RDY-05 workspaceReady → WorkspaceReadyVisual
↓
Domain Models
Model OnboardingPage
id, model, indexLabel, eyebrow, headline, body, metric, command, shaderIntensity, highlights. Static array OnboardingPage.all defines 5 canonical pages.
Enum OnboardingPageModel
encryptedPairing | ideaStudio | promptQueue | reasoningControl | workspaceReady. Drives PageVisualFactory switch.
Model OnboardingFeatureHighlight
title, detail, symbol. Rendered as footer chips per page with staggered entrance animation.
↓
Infrastructure
Repository SwiftDataOnboardingRepository
SwiftData-backed actor implementing OnboardingRepository. Reads/writes OnboardingProgressEntity via ModelContext. Methods: isOnboardingCompleted() → fetch first record; completeOnboarding() → upsert and set isCompleted = true.
DependencyKey OnboardingRepositoryKey
Registers repository in TCA DependencyValues. liveValue = unimplemented (fatalError), testValue/previewValue = mock.
↓
Completion
View HomeView
OnboardingContainerView switches to HomeView when store.isFinished becomes true. Onboarding state is persisted in SwiftData so the flow never repeats.
Key architectural patterns used:

• Parent-Child scoping: OnboardingFlowFeature (navigation + lifecycle) scopes OnboardingPageDemoFeature (page interactions) via Scope(state:action:).
• Dependency injection: Repository is injected at store creation time via withDependencies in OnboardingContainerView.init.
• Factory pattern: PageVisualFactory.make maps domain enum → concrete SwiftUI view, keeping OnboardingFeaturePageView generic.
• Unidirectional data flow: View reads store.state → user action → Reducer processes → state updates → SwiftUI re-renders. No two-way binding outside of TCA.
